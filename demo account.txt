Demo credentials are now managed through shared env files.

Use these files:
- backend/.env.shared
- frontend/.env.shared

Local overrides (machine-specific only):
- backend/.env
- frontend/.env.local

Setup:
1. Run setup-machine.cmd from repo root.
2. Start stack with run-dev.cmd.
3. Use the Auth Switcher demo buttons in the frontend.

Important:
- Keep real/private credentials out of Git.
- Only dummy test credentials belong in .env.shared.
